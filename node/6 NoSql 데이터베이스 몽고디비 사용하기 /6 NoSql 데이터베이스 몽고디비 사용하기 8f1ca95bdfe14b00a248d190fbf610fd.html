6. NoSql 데이터베이스 몽고디비 사용하기

6.1 몽고디비

  • NoSQL
    • 키-벨류타입 : 키를 기준으로 데이터를 조회하고 값으로 데이터를 저장
    • 도큐먼트타입 : json과 유사한 형식의 객체를 담은 데이터를 저장
    • 그래프타입 : 노드를 사용하여 데이터를 저장하고 에지르 사용해 데이터 관계저장
  • 몽고디비는 도큐먼트 타입

6.1.1 데이터베이스 기본 용어

  • 데이터베이스 기본용어
    • 테이블 : 데이터 모음
    • 로우 : 행 데이터 구조항목
    • 컬럼 : 열 자료값
    • 기본키 : 식별 키
    • 외래키 : 테이블 연결키
    • RDB : 관계역 데이터베이스 다른테이블 관계 맺음
    • 스키마 : 테이블 명세 기술 데이터
    • 모델 : 테이블에 있는 컬럼들의 형태를 정의한 클래스
    • 컬랙션 : 몽고디비에서 사용 용어, 도큐먼트의 집합 컬랙션 RDB랑 동일
    • 조인 : 테이블및 컬렉션 조합 기법
    • 트랜잭션 : 작업단위
    • 클러스터 : 처리량을 높일 목적 서버에 저장 기법
    • 샤드 : 큰디비를 작은 단위로 분할하는 기능을 의미 데이터를 작은 단위로 분할하여 붆산시켜 저장함
      • 대규모 데이터베이스를 다루는 시스템에서 성능과 확장성 향상

6.1.2 몽고디비 특징

  • 몽고디비에서 문서는 BSON이라는 데이터 포맷 ‘Binary JSON’의 의미로 JSON을 바이너리 형ㅎ식으로 저장하는 형태
  • JSON에서는 지원하지 않는 자료형인 Date와 바이너리데이터 타입을 지원 Json과 비슷

    1. find()비어 있으면 필터링 없이 모든 데이터를 가져옵니다. 예를 들어 name이
      ‘Andy Park’인 Person 데이터를 찾고 싶을 때는 find({ name ‘Andy Park’})로 검색할 수 있
      습니다.
    1. projection은 결과에 표시할 데이터 필드를 명시합니다.
      1. 예를 들어 Person 쿼리의 결과 중 age를숨기고 싶은 경우 find({}, {name 1, email 1 })처럼 사용할 수 있습니다. 여기서 1은 포함시킨다는 의미
      1. find({}, { age 0 })으로 작성해도 age만 제외할 수 있음
      1. 0을 넣으면 해당 필드를 제외한 필드들을 결괏값으로 사용한다는 의미.
      1. option으로 sort, limit,skip을사용할수 있습니다.
      1. 다른 인수들과마찬가지로JSON 형식로 넣으면 됩니다.
        1 sort: 결과 문서를 정렬할 필드와 정렬 순서(1은 오름차순, -1은 내림차순)를 지정합니다.
        JSON 형식 예 : Person.find({}, {sort: {name ： 1 }})
        2 limit: 결과 문서 수를 지정합니다.
        JSON 형식 예 : Person.find({}, {limit 10} )；
