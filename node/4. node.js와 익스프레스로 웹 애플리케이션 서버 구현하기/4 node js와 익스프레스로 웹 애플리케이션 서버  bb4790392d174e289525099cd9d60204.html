4. node.js와 익스프레스로 웹 애플리케이션 서버 구현하기

4.1 npm 소개

  • 유저가 만든 패키지 등록하는 저장소
  • CLI의미함 노드설치하면 자동임
  • 패키지 많음
  • 패키지 매니저

4.2 패키지와 모듈

  • 노드에서 패키지는 pakage.json으로 정의한 파일 또는 디렉터리를 의미
  • 패키지에는 pakage.json이 포함됨
  1. pakage.json 파일이 있는 디렉토리
  1. 1번을 압축한 파일
  1. 2번을 내려받을수 있는 URL주소
  1. 3번 정보를 가지고 npm 저장소에 <패키지 명>@<버전>으로 등록된 것
  1. 4번을 가리키는 <패키지 명>@<태그>
  1. <패키지명>만 있는 경우는 5번에서 latest 태그를 가리킴
  1. 1번을 결과로 주는 것 URL
  • pakage.json으로 정의한 코드 뭉치가 바로 패키지
  • 모듈은 [node_modules]디렉터리 아래에 있는 파일 또는 디렉터리를 말함
    • [node_modules]에 있는 파일이나 디렉터리는 require()함수로 읽을 수 있음
    • CommenJS는 브라우저, 서버 애플리케이션에서도 모듈 기능을 제공하기 위해 나온 모듈 규약
  • [node_modules]
    • npm install 명령으로 설치한 패키지들이 저장됨
    • 모든 패키지는 모듈
    • npm에 등록하지 않아도 패키지 포맷만 맞는다면 npm install로 설치할 수 있음
  • 패키지만들면 npm install명령으로 설치한 패키지들이 저장
    • 패키지로 만든 코드들은 간단하게 다른 곳에서 설치해 사용이 가능하므로 사설 npm레지스트리에 공통으로 사용하는 패키지를 배포해서 사용
  • require()함수를 사용할 때 단순히 현재 디랙터리의 [node_modules]만 읽는것이 아님

4.3 package.json 파일을 만들기

  • 패키지 만들고 다른 프로젝트에서 설치
  • 패키지 만들려면 pakage.json이 필요함
  • npm init 명령을 사용해 만드는것을 추천
  • 디렉터리 만들고 설정끝

  • index.js 파일도 만들고 사칙연산함수를 추가
  • 실행파일 추가
  1. require() 함수로 모듈을 읽을때 위에서 읽음 그래서 콘솔 실행
  1. module.exports는 require를 사용해 불러왔을 때 반환하는 객체를 저장하는 변수
    1. module은 현재 모듈을 의미
    1. exports는 외부에 노출할 객체를 저장하는변수
    1. 기본값은 비어있음
    1. 여기에서는 사칙연산을 하는 삼수를 객체 타입으로 저장했음
    1. sample-pakage를 설치한 프로젝트에서는 require(’sample-pakage’)로 module.exports 안에 있는 값을 가져옴
  1. 다른 프로젝트에서 설치후 사용
    1. 패키지 설치함 경로 보기
  1. [sample_test]디렉터리에 index.js를 만들고 sample-pakage를 테스트 해봄
    1. require()함수를 사용해 sample-packge를 불러와서 calc 변수에 담음
    1. ~5 sample-package에서 가져온 사칙연산 삼후들을 각각 실행해보고 ㄱ로그를 출력합니다.
    1. 출력결과를 보면 console.log 나옴
      1. 모듈을 가져올 때 파일을 읽는다는 소리임

4.4 패키지 설치, 업데이트, 삭제

  • npm install, npm update, npm uninstall 명령어를 알아보자
  • install
    • npm install 대신 npm i, npm add를 사용가능
      • 패키지명이 중복이 될 수 있음 그것을 방지하기위한 패키지 스코프 사용
        • 패키지 스코프의 경우 네임스페이스 @로 이름이 시작하는데 가장 유명한것은
          • @types
        • 타입스크립트의 타입 정의 패키지는 거의 대부분이 @types 스코프로 시작함
      • name은 패키지명
        • 패키지명 외에도 태그, 버전, 버전범위 필요
        • 업슨경우 latest 태그를 기본값으로 사용
        • npm install에는 사용하는 옵셔을 다음 표에 정리
